Challenge Overview
The age of AI is flooding the internet with synthetic content and automated actors. The future of the web belongs to applications that can distinguish, verify, and empower real humans. World is sponsoring challenges to build "Hyperscale Mini Apps" that leverage the World Network’s unique primitives: World ID (Sybil-resistant identity) and World App (direct access to 38M+ users).



These bounties focus on creating "human-only" digital environments, seamless financial rails, and social experiences that were previously impossible due to bot-farming. Developers will build Mini Apps that live directly inside World App, creating a new paradigm of "Human-to-App" interaction that is instant, verifiably real, and globally accessible.



Ideas

1. The "Sybil-Proof" Social Graph

Challenge Overview: Build a social or community Mini App where every interaction is gated by World ID, ensuring 100% human-to-human engagement without bots or trolls.



Bounty Description: Integration of World ID (Incognito Action); On-chain social actions (voting, tipping, or posting); Zero-gas transaction flow using World’s financial rails; Dashboard showing "Human-only" engagement metrics.



2. Human-Centric DeFi (Universal Access)

Challenge Overview: Create a financial Mini App that utilizes World App’s gas-free transactions to provide "Everyday DeFi" to non-crypto natives.



Bounty Description: Non-custodial lending, savings, or remittance interface; One-tap "Verify & Claim" mechanics for rewards; High-utility UX designed for the 38M+ mobile users; Integration of World’s stablecoin rails.



3. Real-World Asset (RWA) Gateways

Challenge Overview: Build a Mini App that bridges real-world assets or services to verified humans, using World ID to ensure one-person-one-claim.



Bounty Description: RWA distribution or ticketing system; Verifiable proof-of-humanity for asset access; Cross-chain settlement bonus; In-app notification integration for user updates.



4. The "Human-Only" Marketplace

Challenge Overview: Create a peer-to-peer marketplace for goods, services, or data where identity verification eliminates the "trust gap" typical of anonymous web3 commerce.



Bounty Description: World ID-gated merchant and buyer profiles; Escrow or reputation system based on verification history; Mobile-first checkout inside World App; Dispute resolution framework.



5. Proof of Contribution (Human Capital)

Challenge Overview: Design a system for "Work-to-Earn" where humans are rewarded for creative or physical tasks that AI cannot perform.



Bounty Description: Task-verification logic; Automated payout in World-compatible assets; Reputation soulbound tokens (SBTs) for "Proof of Skill"; Integration with FWB’s creative ecosystem.



Submission Requirements



Mini App Integration: Project must be built as a World App Mini App or utilize World ID for core functionality.



Deployment: Must be deployed to World Chain (Mainnet or Testnet).



Human Verification: Must demonstrate a meaningful use of the World ID "Proof of Personhood" (Orb-verified) SDK.



Working Demo: Functional frontend accessible via mobile or the World App simulator.



Open Source: Code must be submitted via a public GitHub repository with a clear README.



Judging Criteria



Humanity & Utility (40%): Does the project solve a real problem for humans? How effectively does it use World ID to prevent bot interference or Sybil attacks?



Technical Execution (30%): Is the code clean and functional? Does it leverage World Chain’s gas-free/low-latency features effectively? Is the Mini App UX "hyperscale ready"?



Viral Potential / Market Fit (20%): How likely is this to gain traction among World App’s 38M+ users? Is the idea original and "World-first"?



Presentation & Demo (10%): Is the problem clearly defined? Does the video demo clearly show the user journey from World ID verification to the final action?
